# Review 介绍视频脚本

对应视频文件：`assets/review-intro-zh.mp4`

## 基本信息

- 语言：中文
- 时长：约 25 秒
- 分辨率：1280 x 720
- 形式：无旁白、动态信息流介绍视频

## 镜头结构

### 1. Hero 开场

画面重点：

- `Review` 标题
- “一个面向 PR、项目与技术文档的证据驱动型评审 skill”
- `PR 审查 / 项目评审 / 上线准备度`

目标：

- 先告诉观众这不是普通 code review 工具

### 2. 核心问题

画面重点：

- PR 能安全合并吗？
- 项目真的完成了吗？
- 现在能发布或提交吗？

目标：

- 用用户真实会问的问题建立共鸣

### 3. 三种范围

画面重点：

- `change`
- `project`
- `artifact`

目标：

- 解释 `Review` 不是只看代码，而是按范围切换评审方式

### 4. 语言专精版

画面重点：

- `TypeScript / React`
- `Python`
- `Java`

目标：

- 强调 `Review` 已经支持按技术栈切换更专业的 code review 视角

### 5. 输出结果

画面重点：

- severity-ordered findings
- completion verdicts
- readiness scorecards
- risk registers
- remediation roadmaps

目标：

- 说明最后给出的不是泛泛评论，而是可执行结论

### 6. CTA 收尾

画面重点：

- 安装命令
- GitHub 地址
- “Evidence-based / Open source / Ready for real review workflows”

目标：

- 让用户知道怎么安装、去哪看、为什么值得试

## 可继续增强的方向

- 增加英文版视频
- 增加配音版
- 增加字幕文件
- 做一版更适合 X / B 站 / 小红书 的竖屏视频
- 加入 GitHub star、release、代码界面的真实录屏片段
